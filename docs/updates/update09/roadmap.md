# Update09 Roadmap

## 目的
Update08までに実装した型編成ナビ、候補トレイ、保存データ連携、部隊編成評価を、実戦で迷わず・軽く・見やすく使えるUI/UXへ整理する。

## 現在の状態
- 状態: Phase 3 実装中（レイアウト/変更ダイアログ実装済み）
- 調査開始HEAD: `779c15c5193949a2122004f2cbbf297f969a8ca9`
- 対象ブランチ: `work`
- Update08最新状態: `Update08.23`（型候補描画キャッシュと重い診断の遅延化まで完了）
- Update09表示ルール: Phaseが分かるように `Update09.x.y` で表示する（`x` はフェーズ番号、`y` は当該フェーズ内の改修回数）。

## 優先テーマ
1. 部隊編成画面と型編成ナビ/型候補一覧の体感性能改善。
2. 部隊編成のレイアウト整理と、武将・装備変更のダイアログ/ポップアップ化の完成。
3. ガイドの最新化と、必要な時だけ読める段階的な説明表示。
4. 補足説明の常時表示を減らし、画面密度を下げる。
5. 型編成ナビ → 型候補一覧 → 候補トレイ → 部隊編成の操作導線を直感化する。
6. PC/スマホそれぞれの表示破綻、はみ出し、操作困難箇所を回帰確認する。

## 完了条件
- 課題ごとにユーザー受け入れ操作を定義し、実装後にPC/スマホ双方で確認する。
- HTMLへ大型ロジックを追加せず、既存責務の外部JS/CSSへ統合する。
- 型検索は候補提示、組み合わせ検討は部隊編成画面という設計を維持する。
- 既存の成立ゲート、保存データ互換、Export/Import互換を維持する。
- Update09の実装内容、HTMLサイズ増減、外部化判断、検証結果、プレビュー同期結果を記録する。

## フェーズ計画

### Phase 1: UI/UX監査と改善案整理（本PR）
- 現行実装とUpdate08.23の残課題を確認する。
- 問題点を分類し、各項目に3案ずつ改善案を作成する。
- 実装順序の推奨を決める。

### Phase 2: 性能・再描画改善
- 部隊編成画面の全体再描画を、操作種類別に必要範囲へ縮小する。
- 型候補一覧と型編成ナビの初回表示・検索・タブ切替・トレイ追加時の再計算を抑制する。
- PC版でのスクロール、固定サイドバー、選択ダイアログ表示時の体感停止を計測・改善する。

### Phase 3: 部隊編成レイアウトと変更ダイアログ完成
- 武将/装備変更をPCでもモーダル/ポップアップ中心に寄せる。
- 編成盤面、選択中スロット、結果サマリー、部隊一覧の優先順位を整理する。
- 候補トレイからの配置先選択を既存選択ダイアログへ自然につなげる。
- 部隊編成のグループは、グループ名と主要ボタンを含めて1行で表示する。グループ名変更は別ダイアログで行い、一覧行を肥大化させない。
- 型選択は重複表示を再考し、同じ選択状態や説明が複数箇所へ出ないようにする。表示型IDはユーザー向け通常表示から外す。
- 部隊のトータルスコアや評価スコアは自動計算結果として扱い、入力ダイアログにはしない。各表示スコアはトータルスコアの下に従属表示する構成へ再考する。
- 履歴へ保存ボタンは不要とし、保存ボタン押下時に現行内容を保存する。マイメモは1行表示に絞り、編集時だけ別ダイアログで編集する。

### Phase 4: ガイド・補足説明・導線整理
- スタートガイド、型編成ナビ、型候補一覧、部隊編成内の文言をUpdate09時点へ最新化する。
- 長い補足は `details`、ヘルプアイコン、モーダルに移し、常時表示は短い行動ラベルへ絞る。
- 全データ表示/保存データ表示の差と、次に押すべきボタンを画面上で明確にする。

### Phase 5: 回帰・プレビュー確認
- JavaScript/JSON/HTML/CSS/ワークフロー検証を実行する。
- PC/スマホの主要導線、保存データImport/Export、型候補からの新規部隊、候補トレイ配置を確認する。
- Push後のpreview同期と公開commit一致を確認する。
