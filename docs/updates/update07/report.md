## Update07.6.1 プレビュー反映不備の原因調査と改修
- 不具合分類: プレビュー配信時の外部JSキャッシュ不整合。
- 根本原因: Update07.6の実装は外部JSへ入っていたが、HTML側の外部JS URLが変わらず、プレビューで旧 `hado_type_candidates.js`、旧 `hado_type_score.js`、旧 `hado_candidate_tray_core.js` が再利用される経路を残していた。
- 影響範囲: 型候補一覧の選択表示・再クリック解除・候補トレイ追加ボタン削除、採点共通処理の主将/副将/補佐/侍従限定句フィルタ。
- 恒久対策: `index.html` と `hado_library_3.0.0.0.html` の外部JS読込URLへ `?v=3.0.0.0-update07.6` を付与し、プレビュー更新時にUpdate07.6の外部JSを再取得させる。
- 再発防止: HTML同一性検証に加え、Update対象外部JSの読込URLが更新版クエリを持つことをローカル監査した。
- HTML増減: `index.html` +231 bytes、`hado_library_3.0.0.0.html` +231 bytes。
- 外部化判断: ロジックはHTMLへ埋め込まず、既存外部JS実装を維持したまま読込URLのみ更新した。
