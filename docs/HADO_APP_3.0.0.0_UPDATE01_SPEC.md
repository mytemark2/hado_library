# 覇道ライブラリ 3.0.0.0 Update 01 仕様固定

## 目的
3.0.0.0では、型検索を「候補を表示する機能」、部隊編成を「組み合わせを検討し、スコアを見ながら完成度を上げる機能」として分離する。

## 型検索の入口
1. 主将から考える
   - 主将リストは既存仕様どおり新武将順で表示する。
   - 主将を選択すると、推奨目的を絞り込む。
   - 推奨目的だけでなく全目的表示へ切替可能にする。
2. 目的から考える
3. 型を直接選ぶ

## 型検索で表示する候補
- 主将
- 副将
- 補佐
- 侍従
- 装備
- 陣形
- 兵器
- 名馬
- 軍馬技能

型検索は候補表示にとどめ、組み合わせを自動決定しない。

## 部隊編成への登録
- 型検索から新規部隊を作成した場合、選択中の型を `evaluationTypeId` に自動設定する。
- 通常の部隊編成画面から作成した場合、評価型を手動指定できる。
- 型を変更しても配置は変更せず、スコアだけを再計算する。
- 配置先未確定の候補は部隊単位の候補トレイへ追加する。

## スコアリング
- 1型につき採点項目は5つ。
- 重み付けは行わない。
- 型充足度は表示しない。
- 技能Lvは採点しない。
- 兵力を採点候補に含める。
- `%`項目は変化率をそのまま加算する。
- `%`でない項目は基準増加量に対する比率×100点で換算する。
- 真偽値は有効100点、無効0点とする。
- `常時スコア` と `条件込み最大スコア` を分けて表示する。
- 異なる型の点数は単純比較せず、同一型の編成改善に使う。

## 部隊編成の保存拡張
- 評価型
- マイメモ
- 候補トレイ
- 保存時点のスコアスナップショット
- 編成履歴
- 最大5グループ
- 1グループ最大12部隊
- 1部隊最大10履歴
- 全体履歴の別上限なし
- 同一内容の連続保存は重複履歴を作らない
- 履歴復元は既存部隊を上書きせず、新規部隊として複製する

## 既存ロジックの扱い
侍従、陣形、兵科、射程、装備重複、参軍、五行、兵器、武装、名馬、軍馬技能の成立判定は既存の部隊編成ゲートを必ず通す。型検索から追加することで既存ゲートを迂回しない。
